# Dice Predictor Analyzer — Bettery Rondo/Twist

Автономное desktop-приложение на Python 3.12+ и PySide6 для сбора истории сумм игры Rondo/Twist на Bettery и построения адаптивного прогноза следующей суммы.

## Что изменилось

- Ручной ввод красного/синего кубика удалён.
- Приложение работает только с итоговыми суммами `2..12`.
- Сбор выполняется через Selenium с сайта `https://bettery.ru/quick-games/game_rondo`.
- История результатов читается только из DOM внутри iframe `quickGame-frame`.
- OCR, OpenCV, скриншоты и распознавание изображений не используются.

## Установка и запуск

```bash
python -m pip install -e .
python -m dice_predictor_analyzer
```

Или после установки:

```bash
dice-predictor-analyzer
```

Selenium 4 использует Selenium Manager и обычно сам подбирает ChromeDriver. Нужен установленный Google Chrome/Chromium.

## Диагностика DOM

При запуске сбора приложение:

1. открывает страницу Bettery Rondo/Twist;
2. ищет iframe `quickGame-frame`;
3. переключается в iframe;
4. пытается найти историю результатов по селекторам из `selectors.json`;
5. пишет в статус найденные результаты и их количество;
6. сохраняет HTML iframe в `debug_page.html`.

Если история не найдена, приложение показывает понятную ошибку/статус, сохраняет HTML и продолжает работать без падения.

## Настройка селекторов

Селекторы вынесены в `selectors.json`, чтобы их можно было менять без правки кода:

```json
{
  "iframe_id": "quickGame-frame",
  "poll_interval_ms": 750,
  "debug_html_path": "debug_page.html",
  "max_results": 300,
  "history_order": "newest_first",
  "result_selectors": [
    "[data-testid*='history' i] div",
    "[class*='history' i] div",
    "[class*='History'] div",
    ".sc-eXVaYZ",
    "div"
  ]
}
```

Классы Styled Components могут меняться, поэтому основной устойчивый механизм — конфигурация селекторов плюс диагностический `debug_page.html`. Если DOM-история на сайте окажется в порядке от старых к новым, поменяйте `history_order` на `oldest_first`.

## Логика обнаружения новых бросков

Приложение каждые 500–1000 мс считывает весь снимок истории из DOM и сравнивает его с предыдущим снимком. Оно не сравнивает только последнюю сумму, поэтому корректно обрабатывает серии вроде `7, 7, 7`.

## Аналитика

До 15 собранных результатов прогноз отключён и показывается сообщение:

```text
Недостаточно данных для анализа.
Собрано X из 15 бросков.
```

После 15 результатов включаются:

- частоты и отклонения от теории;
- давность отсутствия каждой суммы;
- матрица переходов;
- локальные тренды последних 10/20/50 результатов;
- серии одинаковых, чётных/нечётных и диапазонов;
- перекос распределения;
- Adaptive Weighted Ensemble;
- TOP-5 прогнозов;
- сила сигнала;
- диапазоны `2-6`, `7`, `8-12`;
- чётное/нечётное;
- горячие/холодные/отсутствующие числа;
- статистика качества TOP-1/TOP-3/TOP-5;
- журнал последних 100 прогнозов;
- графики последних 50 результатов, силы сигнала, весов и распределения.

## Архитектура

- `collector.py` — `SeleniumCollector`.
- `parser.py` — конфиг селекторов, DOM/HTML parser и сравнение снимков истории.
- `storage.py` — in-memory storage.
- `analyzers.py` — независимые sum-only алгоритмы.
- `weights.py` — adaptive weights engine.
- `forecast.py` — prediction/analytics engine.
- `statistics.py` — качество модели и журнал прогнозов.
- `visualization.py` — Qt-графики.
- `gui.py` — PySide6 dark UI.
